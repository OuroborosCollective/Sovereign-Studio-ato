### 1. TypeScript Error Patterns Fixed

#### Billing Module Type Errors
The billing module had several interconnected TypeScript errors:

| File | Error | Fix |
|------|-------|-----|
| `billingSlice.ts` | Selectors used `state: { billing: BillingState }` | Use `RootState` from store |
| `billingSlice.ts` | Missing `RootState` import | Add `import type { RootState } from '../../store'` |
| `useBilling.ts` | `selectPackages` not imported | Add to imports |
| `costConfig.ts` | `COST_CONFIG` missing export | Add legacy alias |
| `BillingPackage` | Missing `credits` field | Add `credits?: number` |

#### RootState Pattern
```typescript
// ✅ CORRECT - Use RootState from store
import type { RootState } from '../../store';

export const selectCredits = (state: RootState) => state.billing.credits;
export const selectPackages = (state: RootState) => state.billing.packages;

// ❌ WRONG - Anonymous state type
export const selectCredits = (state: { billing: BillingState }) => state.billing.credits;
```

### 2. Test Fixtures - Redux Provider Pattern

**Critical:** Any component using `useSelector` or `useDispatch` MUST be wrapped in a Redux `<Provider>`.

#### Pattern for Test Files
```typescript
import React from 'react';
import { Provider } from 'react-redux';
import { render } from '@testing-library/react';
import { store } from '../../store';  // Adjust path as needed

// Helper function - ADD THIS to every test file
function renderWithProviders(ui: React.ReactElement) {
  return render(<Provider store={store}>{ui}</Provider>);
}

// Then REPLACE all render calls:
render(<SomeComponent />);           // ❌ WRONG
renderWithProviders(<SomeComponent />);  // ✅ CORRECT
```

### 3. Test Assertion Patterns

#### Object Equality in Tests
```typescript
// ❌ WRONG - Strict equality fails with extra fields
expect(result).toEqual({ ok: true, content: 'text', route: '/chat' });

// ✅ CORRECT - Partial matching
expect(result).toMatchObject({ ok: true, content: 'text', route: '/chat' });
```

**When to use `toMatchObject`:**
- Function returns extra fields not in expected object
- Object has dynamic fields (timestamps, UUIDs, etc.)
- Only specific fields need verification

## ⚠️ Common Pitfalls

1. **Redux Provider Missing in Tests**
   - Symptom: "could not find react-redux context value"
   - Fix: Wrap component in `<Provider store={store}>`

2. **Selector Type Mismatch**
   - Symptom: TypeScript errors on selector functions
   - Fix: Use `RootState` instead of anonymous `{ feature: FeatureState }`

3. **Missing Exports**
   - Symptom: "Module has no exported member"
   - Fix: Check both default and named exports, verify import path

4. **Test Assertion Strictness**
   - Symptom: Test fails but code works
   - Fix: Use `toMatchObject` for partial matching

5. **pnpm Corepack Prompt**
   - Symptom: "Corepack is about to download..."
   - Fix: Run `pnpm config set ignore-scripts false` or answer 'Y'
